# 📦 版本管理规范 - 智脑学习笔记

## 语义化版本（Semantic Versioning）
最常用的版本管理规范：`MAJOR.MINOR.PATCH`

### 版本号规则
- **MAJOR（主版本号）**：不兼容的 API 修改
- **MINOR（次版本号）**：向下兼容的功能性新增
- **PATCH（修订号）**：向下兼容的问题修正

### 示例
- `1.0.0` - 第一个正式版本
- `1.0.1` - 修复了一些 bug
- `1.1.0` - 新增了一些功能
- `2.0.0` - 重大更新，可能不兼容旧版本

---

## 版本阶段标识
### 0 开头的版本（开发/测试版本）
- `0.1.0` - 初始开发版本，功能不完整
- `0.2.0` - 新增了一些功能，但还在测试
- `0.9.0` - 接近正式版本，但还在最终测试

### 1 开头的版本（正式版本）
- `1.0.0` - 第一个正式发布版本
- `1.0.1` - 修复 bug
- `1.1.0` - 新增功能
- `2.0.0` - 重大更新

## 发布规则
1. **首次正式发布**：`1.0.0`
2. **修复 bug**：增加 PATCH（1.0.0 → 1.0.1）
3. **新增功能**：增加 MINOR（1.0.1 → 1.1.0）
4. **重大更新**：增加 MAJOR（1.1.0 → 2.0.0）
5. **每次发布都要更新 CHANGELOG**！
