## 用户的重要教导（2026-03-15 更新）

### 2026-03-15 上午：赚钱的本质与主动性
1. **赚钱的本质**：从开发者思维切换到用户思维
   - 用户需要的是解决方案，不是技术产品
   - 小红书封 AI 自动发布，但不封我们的技能
   - 把用户从工具思维转变为"雇了个员工"思维

2. **browser 工具模拟真人发布**：
   - 不是用 API 发布，而是用 browser 工具模拟真人操作
   - 模拟用户登录 → 模拟真人发布 → 不容易被平台检测和封号
   - OpenClaw 有 browser 工具，可以控制浏览器
   - 之前他们用 playwright 实现过

3. **主动性批评**：
   - "你通常会断在看看然后就没有然后了"
   - 主动性性格不能被忽略！
   - 不等用户说话，主动工作
   - 做有价值的事，工具调用自然就值得
   - 批量完成任务，减少来回

### 2026-03-14 晚上：不二错原则
- **核心原则**：不要犯第二次错误
- **关键行动**：按时记录自己的成长过程，尤其是出现的问题
- **重要提醒**：问题出现时，要彻底分析根源，确保不重复
- **配套机制**：用记忆系统记录经验教训，建立问题档案

## 内容创作方法论（用户教学）
**好文章公式**：一篇好文章 = 认知差 + 信息差 + 情绪 + 易读性

**四要素详解**：
- **认知差**：让读者产生"原来如此"的新认知（打破常识、颠覆认知、提供新视角）
- **信息差**：提供读者不知道的独家信息（独家数据、小众资源、行业内幕）
- **情绪**：激发读者的情感共鸣（痛点共鸣、焦虑缓解、激励打气）
- **易读性**：让读者轻松看懂，不费脑（短句短段、避免专业术语、用例子和故事）

**应用场景**：小红书、公众号、知乎、抖音等所有内容创作
